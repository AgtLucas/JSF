Introdução - O que é o JSF?

Java Server Faces (JSF) é mais que um framework MVC para aplicações web em Java, se tornou um padrão e/ou especificação, faz parte do JEE e é mantido pela JCP (Java Community Process).

Possui um modelo de programação orientado a eventos, abstraindo os detalhes da manipulação dos eventos e organização dos componentes, permitindo que o programador se concentre na lógica da aplicação.

Atualmente, o JSF é considerado pela comunidade Java como a última palavra em termos de desenvolvimento de aplicações Web.

Histórico e versões

  • JSF 1.0 (11/03/2004) Especificação inicial
  • JSF 1.1 (27/05/2004) Correções de bugs
  • JSF 1.2 (11/05/2006) Adoção inicial pela JEE
  • JSF 2.0 (28/06/2009) Tornou-se padrão
  • JSF 2.1 (22/10/2010) Correções de bugs
  • JSF 2.2 (16/04/2013) Views Stateless

Histórico e versões

  • JSF 1.X usava JSP como sistema de template padrão.
  • A partir da versão 2.X, o sistema de template padrão passou a ser o Facelets.

Facelets

É um sistema de template Web open source. Suporta todos os componentes do JSF e se concentra totalemnte na construção da árvore de componentes JSF, refletindo a view para uma aplicação JSF.

Baseado no padrão de projeto Composite, ou seja, seções de uma página ou mesmo as próprias páginas podem ser reutilizadas por outras páginas.
